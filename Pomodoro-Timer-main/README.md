# 番茄时钟应用 (Pomodoro Timer)

## 项目概述

本项目是一个基于HarmonyOS开发的番茄工作法计时器应用，帮助用户通过科学的时间管理方法提高工作效率。项目采用模块化设计，具有良好的代码结构和可维护性。

## 功能特性

### 核心功能
- **番茄钟计时**：25分钟工作/5分钟休息循环
- **任务管理**：添加、删除、选择任务
- **数据统计**：可视化展示使用数据
- **个性化设置**：自定义工作/休息时长、提醒方式等

### 页面结构（3个主要页面）
1. **主页面（Index）**：计时器界面和任务列表
2. **统计页面（Statistics）**：数据统计和图表展示
3. **设置页面（Settings）**：应用配置和个性化设置

### 布局组件（多种布局组件）
- **Column**：垂直布局，用于页面主体结构
- **Row**：水平布局，用于导航栏、按钮组等
- **Stack**：堆叠布局，用于计时器圆环效果
- **List**：列表布局，用于任务列表展示
- **Scroll**：滚动布局，用于长内容展示
- **Grid**：网格布局（在统计页面中使用）

### 统计类组件
- **柱状图**：展示最近7天的番茄钟完成情况
- **折线图**：展示专注时间趋势
- **统计卡片**：展示总番茄数、总专注时间、任务总数等关键指标

## 项目结构

```
entry/src/main/ets/
├── pages/              # 页面组件
│   ├── Index.ets       # 主页面
│   ├── Statistics.ets  # 统计页面

├── components/         # 可复用组件
```

## 技术特点

### 1. 组件封装
- **TimerCard组件**：封装计时器显示和控制逻辑，提高代码复用性
- **TaskItem组件**：封装任务列表项，统一任务展示样式

### 2. 工程化规范
- **模块化设计**：按功能划分模块，职责清晰
- **代码注释**：关键函数和类都有详细注释
- **常量管理**：统一管理颜色、配置等常量
- **数据模型**：使用接口和类封装数据结构

### 3. API和Mock数据支持
- **HttpService类**：封装网络请求功能
- **Mock模式**：支持Mock数据，便于开发和测试
- **接口设计**：预留API接口，支持后端对接

### 4. 数据持久化
- 使用`dataPreferences`进行本地数据存储
- 支持任务列表、统计数据、设置等数据的持久化

## 使用说明

### 基本操作
1. **添加任务**：点击"+"按钮，输入任务名称添加新任务
2. **选择任务**：点击任务列表中的任务开始计时
3. **控制计时**：使用"开始/暂停"和"重置"按钮控制计时器
4. **删除任务**：长按任务或向左滑动显示删除按钮
5. **查看统计**：点击顶部"统计"按钮查看数据统计
6. **修改设置**：点击顶部"设置"按钮进行个性化配置

### 统计功能
- 查看总番茄数、总专注时间、任务总数
- 查看最近7天的完成情况（柱状图）
- 查看专注时间趋势（折线图）
- 查看每日详细数据

### 设置功能
- 自定义工作时间和休息时间
- 开启/关闭声音提醒、震动提醒
- 开启/关闭自动开始休息
- 开启/关闭通知显示

## 开发规范

### 代码规范
- 使用TypeScript严格模式
- 遵循HarmonyOS开发规范
- 组件命名采用PascalCase
- 函数命名采用camelCase
- 常量命名采用UPPER_SNAKE_CASE

### 文件组织
- 页面文件放在`pages/`目录
- 可复用组件放在`components/`目录
- 公共工具类放在`common/`目录
- 数据模型放在`models/`目录
